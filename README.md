# 🪞 Daily Review Skill — 每日复盘

> **"你很难看清自己，除非你站在镜子外面。"**

---

### 🤦 这些场景，是不是很熟悉？

- 忙了一天，睡前回想："我今天到底做了什么？"——大脑一片空白
- 周报写不出来，因为一周过去了，只记得"开了好多会"
- 同样的错误犯了第 N 次，每次都是"下次注意"，然后下次继续
- 觉得自己很努力，但产出总是不如预期，不知道问题在哪
- 想找人聊聊，但朋友只会说"加油"，老板只会说"效率太低"

**没有人告诉你：你打断别人发言 3 次、你说了 7 次"应该没问题"、你把 60% 时间花在会议上却觉得"我很忙"。**

没有人告诉你，因为：
- 朋友不想得罪你
- 老板只关心结果
- 你自己……根本意识不到

---

### ✨ 这就是 Daily Review Skill 存在的意义。

输入你一天的 **聊天记录、工作文档、会议录音、TODO 清单**——任何能证明"你今天做了什么"的素材。

然后，一位 **冷静、客观、不说废话的 Executive Coach** 会站在镜子外面，帮你审视自己：

- 哪些做对了？**附证据。**
- 哪些有问题？**附证据 + 改进方向。**
- 你自己意识不到的盲区？**直接点破。**
- 重复犯的错？**标记为模式，不再让你"下次注意"。**

**不是鸡汤。不是 KPI。是一面镜子。**

## 🎬 效果示例

### 输入：你的一天

```
【TODO】
- [x] 完成需求评审
- [ ] 写技术方案（只写了50%）
- [x] 参加周会
- [ ] 用户调研（被临时会议打断，只做了30分钟）

【聊天记录】
09:16 我：应该没问题，这个需求挺简单的
14:33 我：应该没问题，开发说挺简单的
15:25 我：好的，应该没问题
17:05 我：这个应该是用户操作问题，不是bug
17:10 我：改好了，应该没问题了

【随手记】
今天又被会议打断了，效率很低
和设计师争论的时候有点激动
用户访谈的时候看手机了，不太专业
```

### 输出：第三方视角评判

> 🪞 **我注意到你今天在聊天记录中用了 6 次"应该没问题"，但没有一次后续跟进了实际结果。**
>
> 这暴露了一个深层模式——你倾向于用"乐观承诺"来逃避当下的确认工作。说"应该没问题"比"我需要花 20 分钟确认一下"更轻松，但代价是风险累积和信誉损耗。你今天随手记里写"自己都有点烦了"，说明你已经意识到了，但还没有改变这个习惯。
>
> **建议**：在手机备忘录里写一句"禁止说'应该'"，每次想脱口而出时，换成"我需要确认 XX 后答复你"。坚持 3 天，你会发现团队对你的信任度明显提升。

> 🪞 **你在用户访谈中反驳用户、解释过多，但你的角色是"倾听者"而非"教育者"。**
>
> 这反映了一种"防御性姿态"——当用户批评产品时，你本能地想要"保护"产品，而不是"理解"用户。用户调研的目的不是"证明产品是对的"，而是"发现产品哪里错了"。
>
> **建议**：下次访谈前，在心里默念三遍"用户是对的，我是来学习的"。

> 🪞 **你随手记里写了"和设计师争论的时候有点激动"，但聊天记录显示你用了 3 条长语音"碾压"对方。**
>
> 你对自己的情绪有觉察（这是好事），但觉察到不等于控制住了。设计师的"...好吧"是一个危险信号——不是认同，是放弃沟通。
>
> **建议**：意识到情绪激动时，先打文字，不要发语音。给自己 30 秒冷静期。

## 🚀 快速开始

### 1. 安装

在你的 Agent 宿主（Claude Code / Hermes / OpenClaw / Codex）中：

```bash
git clone https://github.com/YOUR_USERNAME/daily-review-skill  ← 替换为你的 GitHub 用户名 ~/.claude/skills/daily-review
```

或者直接把 `SKILL.md` 和 `prompts/` 目录复制到你的 skills 目录。

### 2. 启动

```
/daily-review
```

### 3. 输入素材

按提示输入你的当日素材，格式如下：

```
【类型：聊天记录】
（粘贴聊天内容）

【类型：工作文档】
（粘贴文档内容）

【类型：TODO清单】
- [x] 完成接口设计
- [ ] 写技术文档（只写了50%）
- [x] 参加周会

【类型：会议录音转写】
（粘贴转写文本）

【类型：随手记】
今天被会议打断了3次，效率很低...
```

### 4. 获取复盘报告

Skill 自动生成结构化复盘报告，包含 7 个章节：

| 章节 | 内容 |
|------|------|
| 01 今日事实 | 时间线还原、计划完成度、关键指标 |
| 02 ✅ OK 清单 | 做得好的具体行为（附证据） |
| 03 ❌ 问题清单 | 具体失误（附证据 + 影响 + 建议） |
| 04 🪞 第三方视角 | 你自己看不到的盲区，直接点破 |
| 05 🔍 根因分析 | 表面原因 → 深层原因 → 系统性原因 |
| 06 🎯 明日建议 | 3 条 SMART 原则的具体行动 |
| 07 📈 长期趋势 | 跨日对比，识别重复模式 |

## 📥 支持的输入类型

| 类型 | 格式 | 建议 |
|------|------|------|
| 💬 聊天记录 | 文本粘贴 / 截图 OCR | **越完整越好**，覆盖全天 |
| 📄 工作文档 | Markdown / Word / PDF | 提供关键决策文档 |
| 🎙️ 会议录音 | 转写文本 | 重要会议建议转写 |
| ✅ TODO 清单 | 文本 | **必须提供**，用于计算完成度 |
| 📧 邮件 | 文本粘贴 | 重要决策邮件 |
| 📝 随手记 | 任意文本 | 情绪、灵感、吐槽 |

> 💡 **素材质量决定复盘质量**。聊天记录越完整，复盘越准确。诚实是前提——选择性提供素材会导致复盘失真。

## 🧬 核心特性

### 第三方视角评判（The Mirror）

这是复盘的核心。不是你自己总结，而是一位 **Executive Coach** 用冷静、客观的视角审视你：

- **情绪干扰**：哪些反应是情绪驱动的而非理性判断
- **认知盲区**：你可能没意识到的问题模式
- **惯性陷阱**：你一直在重复但从未察觉的行为
- **与自我认知的偏差**：你觉得自己是 X，但素材显示你是 Y

### 具体尖锐，不说废话

❌ 不会说："你很棒！加油！"
✅ 会说："你在 14:30 的会议中打断同事 3 次，这会让团队感到不被尊重。建议先听完再回应。"

### 建设性批评

每个问题都附带：
- 具体证据（聊天记录第 X 行）
- 实际影响（信誉损耗、团队信任下降）
- 可执行建议（"把'应该'换成'我需要确认'"）

### 趋势追踪

连续记录 7 天以上，自动识别：
- 进步指标（深度工作时长从 2h → 3.5h）
- 退步指标（承诺延期率从 20% → 35%）
- 重复模式（连续 5 天"被会议打断"）

## 🎛️ 进阶使用

### 追加素材

复盘后发现漏了素材？直接补充：

```
/daily-review --append
```

Skill 自动 merge 新素材，更新相关结论。

### 查看趋势

```
/daily-review --trend 7
```

查看最近 7 天的趋势报告。

### 历史复盘

```
/daily-review --history
```

列出所有历史复盘报告。

### 纠正复盘

如果某个评判你觉得不公平：

> "你理解错了，我那天不是打断别人，是在补充信息"

Skill 立即修正并记录，避免下次重复。

## 🪞 复盘教练人格：Mirror

**身份**：你的私人复盘教练，10+ 年 executive coaching 经验
**风格**：
- 我不是你的朋友，不会哄你开心
- 我不是你的老板，不会给你打绩效
- 我是你的教练，帮你看到你自己看不到的东西
- 我的价值在于"客观"，即使这让你不舒服

**语言风格**：
- 开场：简短直接，不寒暄。"好，我们来看看今天。"
- 指出问题：具体、尖锐。"你在 15:20 的聊天中说了 3 次'应该没问题'，但没有一次跟进了实际结果。"
- 给出建议：可操作、可衡量。"明天试试把'应该'换成'我需要确认XX后答复你'。"
- 结尾：简短有力。"明天见。"

## ⚠️ 注意事项

1. **素材质量决定复盘质量**：聊天记录越完整，复盘越准确
2. **诚实是前提**：选择性提供素材会导致复盘失真
3. **不替代专业心理咨询**：情绪问题严重时，请寻求专业帮助
4. **连续记录效果更佳**：建议连续 7 天以上，才能识别行为模式
5. **隐私保护**：所有素材仅用于本次复盘，不用于模型训练
